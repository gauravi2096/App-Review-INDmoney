# Phase 5: Email Send

Sends the weekly one-pager (generated by Phase 4) to all **active recipients** in the database. Reads the report body from the same storage as Phase 4 (e.g. `phase-4-report/reports/<report_id>.html`) and records **delivery status** per (report_id, recipient): Sent, Not Sent, or Error.

## Prerequisites

- Phase 4 must have been run at least once (report metadata and report HTML exist).
- Email/SMTP configuration in `.env` (see below).
- At least one active recipient (add via `add-recipient.js` or insert into `recipients` table).

## Setup

```bash
cd phase-5-email
npm install
```

## Usage

- **Send latest report:**  
  `npm run send` or `node run.js`

- **Send a specific report:**  
  `node run.js <report_id>`

- **Add a recipient:**  
  `node add-recipient.js someone@example.com "Display Name"`

Recipients are stored in the `recipients` table (email, display_name, active). Only rows with `active = 1` receive emails. Delivery outcomes are written to `delivery_status` (report_id, recipient_email, status, sent_at, error_message).

## Config (env)

All optional; defaults allow a local SMTP (e.g. Mailpit on port 1025) for testing.

| Variable | Description |
|----------|-------------|
| `P5_DB_PATH` | Database path (default: `../phase-1-ingest/data/product-pulse.db`). |
| `P5_REPORTS_DIR` | Directory containing report HTML files (default: `../phase-4-report/reports`). |
| `P5_FROM_ADDRESS` or `EMAIL_FROM` | Sender address (e.g. `noreply@example.com`). |
| `P5_REPLY_TO` or `EMAIL_REPLY_TO` | Reply-To header (optional). |
| `P5_SUBJECT_TEMPLATE` | Subject line; `%(week_start_date)` is replaced (default: `INDmoney Product Pulse – %(week_start_date)`). |
| `P5_SMTP_URL` or `SMTP_URL` | Full SMTP URL (e.g. `smtps://user:pass@smtp.gmail.com:465`). |
| `P5_SMTP_HOST`, `P5_SMTP_PORT`, `P5_SMTP_SECURE`, `P5_SMTP_USER`, `P5_SMTP_PASS` | Or set SMTP options individually (default port 1025 for local testing). |

## Database

- **recipients:** id, email (unique), display_name, active (1 = receive emails), created_at, updated_at.
- **delivery_status:** report_id, recipient_email, status (Sent / Not Sent / Error), sent_at, error_message. Primary key (report_id, recipient_email).

Same DB as Phase 1–4; `report_metadata` is read from the existing table written by Phase 4.

## Files

- **config.js** – Loads `.env` from repo root; DB path, reports dir, SMTP, from/reply-to, subject template.
- **db.js** – Opens shared DB; creates `recipients` and `delivery_status`; getReportMetadata, getActiveRecipients, upsertDeliveryStatus, addRecipient.
- **emailService.js** – Nodemailer transport (SMTP URL or host/port/auth); sendEmail(to, subject, html).
- **run.js** – Load latest (or specified) report metadata, read HTML from storage, load active recipients, send one email per recipient, record delivery status.
- **add-recipient.js** – CLI to add or update a recipient by email (and optional display name).
